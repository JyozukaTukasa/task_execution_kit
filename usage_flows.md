# タスク実行用の使い方

## 全体の流れ

| 手順 | 何をするか | 出力 |
|---|---|---|
| 1 | 割り振られたタスクを貼る | 元タスク |
| 2 | 目的と完了条件を整理する | タスク理解メモ |
| 3 | 前後工程と確認先を整理する | 不明点一覧 |
| 4 | 自分がやる小タスクに分解する | 自分用 WBS |
| 5 | 1日単位の進め方に直す | 今日やること |
| 6 | 固定フォーマットで報告する | 作業報告 |

## 進め方

1. 割り振られたタスクの内容をそのまま貼る  
2. そのタスクで何を作れば終わりかを書く  
3. このタスクの前に何があり、次に誰が使うかを書く  
4. 分からないことを質問として分ける  
5. 自分が今すぐ着手できる小タスクに分ける  
6. 作業後は `work_report_template.md` の形で報告する  

## 小さいタスクしか見えていない時

次の 4 つを必ず確認する。

1. このタスクは何のためにあるか  
2. 終わったら誰に渡すか  
3. 先に必要な情報や資料はあるか  
4. どこまでできたら「終わり」と言えるか  

## AI への基本指示

```text
このタスクを自分で進められるように分解してください。
目的、成果物、完了条件を整理してください。
前後工程が分からない場合は、確認すべきこととして分けてください。
自分が今すぐ着手できる小タスクを順番に並べてください。
```

## 迷った時のルール

1. 分からないことを推測で埋めない  
2. まず確認事項を作る  
3. 小タスクは 30分から半日くらいで終わる粒度にする  
4. 「今すぐできること」から並べる  
5. 報告は自由文ではなく固定フォーマットで書く
